# Open Source GitHub Repositories for AI-Powered Job Application Automation

**The landscape for building job application automation systems has matured significantly in 2024-2025, with Python-based solutions achieving remarkable sophistication.** The most notable repository, AIHawk, has garnered **29,000+ stars** and features AI-powered resume customization, while modern agent frameworks like CrewAI (40,000+ stars) and PydanticAI (10,000+ stars) now offer production-ready orchestration capabilities that rival commercial solutions. Combined with specialized tools for web scraping (JobSpy supporting 8+ job boards), stealth automation (SeleniumBase with anti-detection), and ATS optimization, developers can now build comprehensive systems that handle the entire application workflow from job discovery through submission.

This ecosystem represents a fundamental shift from simple form-filling scripts to intelligent, multi-agent systems. The convergence of three technology waves—LLM-powered agents, undetected browser automation, and multi-platform job scrapers—has made truly autonomous job application systems technically feasible. The repositories identified here are actively maintained (all updated in 2024-2025), production-tested, and designed to work together through standard Python interfaces.

## Job application automation bots lead with sophisticated AI integration

The most mature category contains complete, end-to-end automation solutions. **AIHawk (feder-cr/Jobs_Applier_AI_Agent_AIHawk) dominates with nearly 29,000 stars** and has been featured in major media outlets including TechCrunch, Wired, and The Verge. This repository offers intelligent job search with customizable criteria, dynamic resume generation tailored to each position, AI-powered personalization for application questions, and bulk application capability with quality control. The system integrates GPT models for natural language understanding and includes smart filtering with company blacklisting. Despite third-party provider plugins being removed due to copyright concerns, the core architecture remains fully open source and highly functional.

**GodsScion's Auto_job_applier_linkedIn (997 stars) claims to apply to 100+ jobs in under an hour** while maintaining human-like behavior patterns. Updated as recently as December 2024 (version 24.12.29.12.30), this Python 3.10+ solution uses Selenium with undetected-ChromeDriver to bypass LinkedIn's bot detection. Key innovations include customized resumes based on job information, AI-powered resume and cover letter generation via OpenAI integration, humanized clicks and mouse movements, and a Flask-based UI dashboard for managing applications. The repository has gained recognition from Jason Huggins (creator of Selenium) and Jeff Hammerbacher (cofounder of Cloudera), signaling strong technical credibility. Notable features include screenshot debugging for failed applications, continuous 24/7 operation mode, and comprehensive Excel export logging.

The lookr-fyi bot (updated July 2025, v1.8.0) takes a privacy-first approach by using **Ollama AI for on-device processing**, eliminating the need to send resume data to external APIs. This repository emphasizes ATS-optimized resume generation, personalized cover letter creation, and iterative learning that typically requires ~10 reviews before running autonomously. The system shows AI reasoning transparency by displaying decision-making processes and includes references for every answered question. For users concerned about data privacy, this represents the most secure option while maintaining sophisticated AI capabilities.

For multi-platform support, **EasyApplyJobsBot (wodsuz) extends beyond LinkedIn to support Glassdoor, AngelCo, Greenhouse, Monster, GlobalLogic, and Djinni**. This repository offers both free and Pro versions, with the Pro tier adding AI-powered question answering and headless browser operation. The bot implements time breaks to prevent detection thresholds and allows following/unfollowing companies upon application. The commercial support model (www.automated-bots.com) provides paid tech support and documentation packages, making it suitable for users who need reliable assistance.

For developers targeting major Applicant Tracking Systems directly, simonfong6's auto-apply specifically handles **Greenhouse, Lever, Workday, and Jobvite**—the four platforms used by most technology companies. Rather than targeting job boards, this Python/Selenium solution applies directly through company career pages, often bypassing additional screening layers. Similarly, whatamithinking's japplr focuses on ZipRecruiter and Monster with configurable global and local search filters, scheduled runs (e.g., every 15 minutes), and quantity control per search.

## Modern AI agent frameworks enable sophisticated orchestration

The agent framework landscape has evolved dramatically with several production-ready options emerging in 2024-2025. **CrewAI (40,000+ stars) offers the most intuitive approach for job application systems through role-based agent collaboration**. Built completely independent of LangChain, CrewAI features a dual architecture: Crews for autonomous agent collaboration and Flows for event-driven, deterministic workflows. The framework supports sequential and hierarchical process orchestration, human-in-the-loop workflows, and built-in tools with custom tool integration. Benchmarks show CrewAI runs **5.76x faster than LangGraph**, while supporting multiple LLM providers including OpenAI, Anthropic Claude, Google Gemini, and local models via Ollama. For job applications, developers could create specialized agents for resume parsing, job matching, application tracking, and cover letter generation, with role-based agents (Researcher, Writer, Applicant, Tracker) working collaboratively.

**PydanticAI (10,000+ stars) takes a radically different approach by emphasizing type safety and structured outputs**. Built by the Pydantic team—whose validation library underlies OpenAI SDK, Anthropic SDK, and LangChain—this framework guarantees output validation with Pydantic models and leverages Python type hints for IDE autocomplete and static checking. Key features include dependency injection for type-safe context passing, durable execution for handling long-running workflows with state preservation, and streaming support for continuous structured output. The framework supports an extensive range of providers including OpenAI, Anthropic, Google Gemini, DeepSeek, Grok, Cohere, Azure AI Foundry, Amazon Bedrock, and more. For job applications, PydanticAI excels at creating type-safe data structures for job listings and resumes, validating parsed job requirements, managing database connections through dependency injection, and implementing tool approval for sensitive actions like submitting applications.

Microsoft's unified Agent Framework (5,000+ stars) combines the best of AutoGen and Semantic Kernel with multi-language support for Python and .NET. The framework features graph-based workflows with DAG orchestration, streaming, checkpointing, and time-travel debugging. The included DevUI provides an interactive developer interface for testing and debugging, while built-in OpenTelemetry integration enables production observability. Multiple agent patterns are supported including Sequential, GroupChat, Concurrent, Magentic, and Handoff. For enterprise environments requiring Azure integration, security compliance, and scalable deployment, this represents the most mature option.

**OpenAI's official Agents SDK (5,000+ stars, newly released) emphasizes lightweight design with provider-agnostic support for 100+ LLMs via LiteLLM**. Despite the OpenAI branding, the framework works with Anthropic, Google, Cohere, and others through a unified interface. Key capabilities include agent handoffs for transferring control between specialized agents, guardrails for input/output validation, session memory with automatic conversation history (SQLite or Redis), and Temporal.io integration for durable, long-running workflows with human-in-the-loop support. The minimal abstraction approach provides maximum flexibility while maintaining simplicity.

**Agno (formerly PhiData, 10,000+ stars) claims extreme performance with 529x faster agent instantiation than LangGraph and 24x lower memory consumption**. The framework includes AgentOS runtime, a production-ready FastAPI app with integrated UI, and runs entirely in your cloud for privacy. Notable features include multi-agent teams under shared leadership, step-based deterministic workflows, best-in-class Model Context Protocol (MCP) integration, and built-in memory with RAG support for 20+ vector stores. The framework provides 100+ toolkits with thousands of pre-built tools. For job applications requiring real-time job matching, monitoring UI, candidate profile memory, and resume-to-job-description matching via RAG, Agno offers compelling performance advantages.

MetaGPT (44,000+ stars) and CAMEL (6,000+ stars) round out the frameworks with research-oriented approaches. MetaGPT uses an SOP-based "Code = SOP(Team)" philosophy to simulate software company workflows with role-based agents (product managers, architects, engineers, testers), while CAMEL focuses on large-scale agent systems supporting up to 1 million agents for studying scaling laws. Both frameworks have strong academic backing—MetaGPT's AFlow paper was accepted at ICLR 2025 ranking #2 in the LLM-based Agent category, while CAMEL has 100+ researchers dedicated to multi-agent systems.

## Supporting libraries provide critical building blocks

**JobSpy (speedyapply) emerges as the most comprehensive job scraping solution, supporting LinkedIn, Indeed, Glassdoor, Google, ZipRecruiter, Bayt, Naukri, and BDJobs with concurrent scraping across multiple platforms**. Updated in 2024-2025 for Python 3.10+, the library returns pandas DataFrames with comprehensive filtering including job type, remote work, hours_old, and easy_apply status. The tool supports 50+ countries for Indeed and Glassdoor, implements Google-specific search syntax, and includes proxy support essential for LinkedIn scraping. Indeed has no rate limiting making it the most reliable source, while LinkedIn rate limits around the 10th page requiring proxy rotation. Installation is simple via `pip install -U python-jobspy`, and the API accepts site lists, search terms, locations, and result limits. The library enforces annual salary conversion and returns detailed company information for Indeed-specific scrapes.

JobFunnel (PaulMcInnis) provides a complementary approach focused on long-term job search management with **no duplicate jobs across scrapes and comprehensive status tracking** (interested, applied, interview, offer). The Python 3.11+ tool supports Canada, USA, UK, France, and Germany with company block lists, job age filtering, remote work filtering by remoteness level, and crontab automation support. The YAML-based configuration system produces CSV output with master lists and includes data recovery from cache folders. Development features include pre-commit hooks with Black, isort, and Flake8, plus pytest test suite with coverage.

For enterprise-focused ATS job boards, adgramigna's job-board-scraper targets **Greenhouse, Lever, Ashby, and Rippling with production deployment** at levergreen.dev. The architecture uses Scrapy spiders with unified data models, PostgreSQL storage, dbt Core for data transformation, and daily automated scraping via GitHub Actions. HTML caching in S3 prevents throttling, while Hightouch handles reverse ETL. This repository demonstrates scalable architecture patterns suitable for production systems.

**SeleniumBase (seleniumbase) stands out as the most comprehensive form automation solution with UC Mode and CDP Mode for stealth operation and bypassing bot detection**. This all-in-one framework includes automatic driver management, multi-browser support (Chrome, Edge, Firefox, Safari), built-in waiting mechanisms, dashboard and reporting, and 200+ examples. The library handles automatic CAPTCHA solving in UC/CDP modes, provides form filling with wait-for-ready, supports proxies with authentication, and offers user-agent customization. Installation requires only `pip install seleniumbase`, with pytest command-line options for browser selection, headless execution, visual debugging, proxy configuration, mobile emulation, and undetected mode. For job applications requiring stealth capabilities and reliability, SeleniumBase represents the most mature option.

Microsoft's playwright-python provides a modern alternative with official support for Chromium, Firefox, and WebKit across Linux, macOS, and Windows. The async/sync APIs include modern auto-waiting mechanisms, network interception, mobile emulation, and video recording capabilities. Playwright offers better performance than Selenium for complex single-page applications and modern web frameworks, with ever-green browser versions and built-in trace viewers for debugging. The Tahanima boilerplate repository provides a ready-to-use test automation architecture with Page Object Model structure, pytest integration, and production-ready patterns.

For resume processing, **pyresparser (OmkarPathak) offers the most straightforward parsing solution** with spaCy and NLTK support for extracting names, emails, mobile numbers, skills, total experience, education, designations, and company names from PDF and DOCX files. The CLI interface can process entire directories, returns structured dictionaries, exports to JSON, and supports custom regex patterns and skills CSV files. Installation is simply `pip install pyresparser` with documentation at omkarpathak.in/pyresparser. For resume generation, hanula's resume tool generates resumes from YAML to PDF and HTML using PyYAML, Jinja2, Markdown, and WeasyPrint with multiple themes and configurable PDF generation.

**ATS optimization tools include panasenco's resume-optimizer which uses OpenAI GPT to rewrite work highlights with ATS keywords** extracted from job descriptions. The tool works with JSON Resume format, accepts job descriptions from files or clipboard, and operates via CLI for automation. For interactive feedback, miteshgupta07's ATS-Scoring-System provides a Streamlit web interface with resume parsing (PDF, DOCX, TXT), entity extraction using trained spaCy models, keyword-based scoring, improvement suggestions, and visual entity display.

## Recommended implementation architecture combines best-of-breed tools

For developers building production job application systems, the optimal architecture combines **CrewAI for agent orchestration, JobSpy for multi-platform job discovery, SeleniumBase for stealth automation, and resume-optimizer for ATS compliance**. This stack provides the best balance of functionality, performance, and maintainability while remaining entirely Python-based with standard interfaces.

The data flow would begin with JobSpy scraping LinkedIn, Indeed, and Glassdoor concurrently, filtering by job type, remote work, and recency. A CrewAI Job Researcher agent analyzes these listings against candidate requirements using a RAG system built with Agno's vector store integration. The Resume Tailor agent uses resume-optimizer to extract ATS keywords from job descriptions and generates customized resumes using hanula's YAML-to-PDF generator. The Cover Letter Writer agent creates personalized cover letters through GPT integration. Finally, the Application Manager agent uses SeleniumBase in UC Mode to submit applications with human-like behavior patterns, while the Tracking agent monitors responses and schedules interviews.

For performance-critical applications requiring the fastest agent instantiation, **Agno's 529x speed advantage over LangGraph** becomes significant when processing hundreds of jobs daily. For type-safety requirements in enterprise environments, PydanticAI ensures all data structures remain validated throughout the pipeline. For Azure-native deployments, Microsoft Agent Framework provides the deepest ecosystem integration with enterprise security compliance.

## Conclusion: Mature ecosystem enables autonomous job search

The convergence of sophisticated job application bots, production-ready AI agent frameworks, and specialized supporting libraries has created an ecosystem where truly autonomous job application systems are now technically feasible with entirely open-source components. The most striking finding is the maturity level—repositories like AIHawk with 29,000 stars and media coverage, CrewAI with 40,000 stars and benchmark-proven performance, and JobSpy supporting 8+ major job boards represent production-grade software rather than proof-of-concepts.

Three key insights emerge from this research. First, **anti-detection capabilities have become table stakes**—every mature automation tool now includes stealth features like undetected ChromeDriver, randomized human-like behavior, and CAPTCHA handling. LinkedIn and other platforms actively detect automation, making tools without these capabilities effectively unusable. Second, **AI integration has shifted from optional to essential**—the top repositories all use LLMs for resume customization, question answering, and intelligent filtering rather than simple template matching. Third, **the framework landscape has fragmented by use case** rather than consolidating, with CrewAI optimizing for role-based collaboration, PydanticAI for type safety, Agno for performance, and Microsoft Agent Framework for enterprise compliance.

For developers familiar with Agno AI and LangGraph, the ecosystem now offers direct alternatives (CrewAI, OpenAI Agents SDK) and complementary tools (JobSpy, SeleniumBase) that integrate seamlessly through standard Python interfaces. The combination of these repositories enables building systems that can autonomously discover jobs, customize applications, optimize for ATS, submit with stealth, and track outcomes—transforming job search from manual labor into an orchestrated multi-agent workflow. The technical capability exists today; the remaining challenges are ethical considerations around automation, platform terms of service compliance, and ensuring applications maintain quality and authenticity despite automation.
